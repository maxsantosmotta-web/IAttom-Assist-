1. CORRIGIR TIKTOK OAUTH — REDIRECT URI
━━━━━━━━━━━━━━━━━━━━



Problema identificado:
O OAuth TikTok está montado corretamente, usando client_key, scope, response_type=code e state.
A falha provável/diagnosticada está no redirect_uri salvo em tiktok_config.redirect_uri, que pode estar diferente do redirect_uri cadastrado no TikTok Developers.

Correção:

Garantir que o OAuth TikTok use sempre o redirect_uri salvo em tiktok_config.redirect_uri.

No painel ADMIN da integração TikTok, exibir claramente o redirect_uri atual usado pelo sistema.

Adicionar campo editável para o admin salvar manualmente o redirect_uri correto.

Não sobrescrever automaticamente com window.location.origin se já existir redirect_uri salvo.

Evitar salvar domínio temporário .replit.dev quando a produção usa .replit.app.

Exibir alerta textual no ADMIN:
“Este redirect_uri deve ser idêntico ao cadastrado no TikTok Developers.”

Manter client_key, client_secret, scope=user.info.basic, response_type=code e state como já estão.

Após salvar, o botão “Conectar TikTok” no USER deve usar exatamente esse redirect_uri salvo.


Critério de sucesso:
Ao clicar em “Conectar TikTok”, a autorização real do TikTok deve abrir sem erro de client_key/redirect_uri.

2. CORRIGIR CHECKLIST TIKTOK — FALSO POSITIVO
━━━━━━━━━━━━━━━━━━━━

Problema identificado:
O checklist ADM mostra “Conectado ativo” quando o admin apenas salva clientKey + clientSecret, porque isActive é definido como true no save das credenciais.

Correção:

Remover a lógica que seta isActive=true automaticamente ao salvar credenciais TikTok.

Status correto:

Pendente: sem clientKey ou sem clientSecret.

Configurado: clientKey + clientSecret salvos, mas sem OAuth concluído/token salvo.

Conectado ativo: somente se existir access_token TikTok válido salvo após OAuth real.


O checklist deve verificar conexão real em user_tiktok_connections ou tabela equivalente de tokens/conexões TikTok.

Enquanto o OAuth não concluir com sucesso, TikTok deve aparecer como “Configurado”, não “Conectado ativo”.


Critério de sucesso:
TikTok só aparece “Conectado ativo” depois do usuário concluir OAuth real.

3. CORRIGIR MERCADO LIVRE — TOKENMANAGER APÓS RESTART
━━━━━━━━━━━━━━━━━━━━

Problema identificado:
Mercado Livre funciona, OAuth salva tokens no DB e refreshMLToken existe.
O erro está no TokenManager in-memory, que perde os tokens após restart/deploy e não reidrata os tokens salvos no DB.

Correção:

Criar rotina de startup para carregar tokens válidos do DB ao iniciar o servidor.

Re-registrar no TokenManager:

tokens por usuário de userMlConnections, se existirem e estiverem ativos


Se access_token estiver expirado, usar refresh_token salvo para renovar automaticamente.

Persistir novo access_token, refresh_token e expiresAt/tokenExpiry no DB após refresh.

Não alterar o fluxo OAuth atual do Mercado Livre.

Não alterar tela USER/Admin do Mercado Livre, exceto se necessário para refletir status real.


Critério de sucesso:
Depois de restart/deploy, Mercado Livre continua reconhecendo tokens salvos e consegue renovar sem exigir nova conexão manual.
